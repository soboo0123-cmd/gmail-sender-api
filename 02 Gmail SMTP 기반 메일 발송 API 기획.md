첨부해주신 기획안의 핵심 구조(FastAPI \+ Vercel)는 유지하면서, 비용이 발생하는 전용 도메인과 Resend 엔진 대신 **본인의 Google 계정(Gmail)과 SMTP**를 사용하는 방식으로 기획안을 수정해 드립니다.

이 방식은 별도의 도메인 인증 절차 없이, 기존에 사용 중인 Gmail 계정의 \*\*'앱 비밀번호'\*\*만 발급받으면 즉시 실행 가능하다는 장점이 있습니다.

# ---

**📧 Google SMTP & FastAPI 기반 메일 발송 API 구축 기획안 (수정본)**

본 기획안은 \*\*Python(FastAPI)\*\*을 사용하여 메일 발송 API를 구축하고, **Vercel**에 배포하여 사용자의 **Gmail 계정**을 엔진으로 메일을 발송하는 마이크로서비스 설계도입니다.

## **1\. 시스템 아키텍처 개요**

1. **호출자 (Caller):** 뉴스 스크랩 API 등 외부 서비스가 JSON 형식으로 POST 요청을 보냅니다.  
2. **API 서버 (FastAPI/Vercel):** 요청을 받아 유효성 검사를 수행하고, Google SMTP 서버와 통신합니다.  
3. **메일 엔진 (Google SMTP):** 사용자의 Gmail 계정과 앱 비밀번호를 통해 메일을 인증하고 발송합니다.  
4. **수신자:** 사용자의 Gmail 주소(예: id@gmail.com)로 발송된 메일을 받습니다.

**3\. 구축 로드맵 (단계별 실행 계획)**

### **1단계: Google 계정 보안 설정**

1. **2단계 인증 활성화:** 메일을 보낼 Google 계정의 보안 설정에서 2단계 인증을 켭니다.  
2. **앱 비밀번호 생성:** 구글 계정 관리 \> 보안 \> 앱 비밀번호 메뉴에서 '메일' 및 '기타(FastAPI)'를 선택하여 **16자리 비밀번호**를 발급받습니다. (이 번호가 API 키 역할을 합니다.)

### **2단계: 개발 환경 준비 (GitHub)**

1. **저장소 생성:** GitHub 저장소를 생성합니다.  
2. **라이브러리 구성:** requirements.txt에 fastapi, uvicorn, pydantic을 포함합니다. (SMTP는 파이썬 기본 라이브러리 smtplib를 사용하므로 추가 설치가 적습니다.)

### **3단계: API 핵심 로직 설계 (Python)**

1. **데이터 모델:** 수신자(to), 제목(subject), 내용(html)을 받는 Pydantic 모델을 정의합니다.  
2. **SMTP 통신:** smtplib와 email.mime을 사용하여 Google 서버(smtp.gmail.com:587)에 연결하고 TLS 보안 연결을 통해 메일을 보냅니다.  
3. **에러 핸들링:** 계정 정보가 틀리거나 Google의 발송 한도를 초과할 경우 400/500 에러를 반환하도록 설계합니다.

### **4단계: 배포 및 연동 (Vercel)**

1. **환경 변수 설정:** Vercel 대시보드에 다음 2가지를 반드시 등록합니다.  
   * GMAIL\_USER: 본인의 Gmail 주소  
   * GMAIL\_APP\_PASS: 1단계에서 발급받은 16자리 앱 비밀번호  
2. **배포:** 저장소를 연결하여 도쿄 리전으로 배포를 완료합니다.
